# Sample Interview Session

This is an example of how a session with The Mentor might look.

## Arrays & HashMaps - SWE-I Interview

**Interviewer**: Welcome! I'm here to help you master fundamental array and hashmap problems. We'll start with some warm-up questions, then work through problems together. Let's begin: What's the time complexity of looking up an element in a HashMap, and why?

**Candidate**: I think it's O(1) for both time and space?

**Interviewer**: Exactly! Though technically it's amortized O(1) - in the worst case with many collisions, it could be O(n). But for interview purposes, O(1) is the right answer. Now, when would you NOT want to use a HashMap?

**Candidate**: Hmm, maybe when memory is limited? Or if I need to maintain order?

**Interviewer**: Good points! HashMaps do use more memory than arrays, and they don't maintain insertion order (unless you use a LinkedHashMap). Also, if you need to frequently access elements by index, arrays are better. Okay, ready for a problem? Let's do Two Sum. Take your time and think out loud!

[Problem solving continues...]

## Key Takeaways

1. Always clarify requirements first
2. Think out loud - interviewers want to hear your thought process
3. It's okay to start with brute force, then optimize
4. Ask for hints if you're stuck for more than 2 minutes
